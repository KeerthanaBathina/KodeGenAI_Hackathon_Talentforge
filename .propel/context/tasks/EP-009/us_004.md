# US-004 — Bulk Requisition CSV Import with Validation and Error Report

## User Story

**As a** recruiter,  
**I want** to upload a CSV file to bulk-import multiple job requisitions at once, with clear validation errors reported per row,  
**So that** new hiring cycles can be set up in minutes rather than hours of manual data entry.

## Business Value

Bulk import reduces onboarding time for new hiring cycles, particularly during peak periods when 20+ requisitions need to be opened simultaneously. Per-row error reporting prevents silent data quality issues.

## Acceptance Criteria

### Scenario 1: Valid CSV imported and requisitions created
**Given** a recruiter uploads a CSV with 10 valid requisition rows  
**When** the import processes  
**Then** 10 `requisitions` records are created with status `"open"` and a success summary reads "10 requisitions imported successfully."

### Scenario 2: Invalid rows reported without blocking valid rows
**Given** a CSV with 8 valid rows and 2 rows with missing required fields  
**When** the import processes  
**Then** 8 requisitions are created; a downloadable error report lists row numbers, column names, and validation messages for the 2 invalid rows.

### Scenario 3: CSV column headers validated before processing
**Given** a CSV is missing the required `role_title` column  
**When** the file is uploaded  
**Then** the import is rejected immediately with "Missing required column: role_title" and no rows are processed.

### Scenario 4: Import is idempotent with duplicate detection
**Given** the same CSV is uploaded twice  
**When** the second upload processes  
**Then** duplicate requisitions (same role_title + department + location) are skipped with a "duplicate" note in the error report; no duplicate records are created.

## INVEST Checklist

- [x] Independent — import tool independent of manual requisition creation
- [x] Negotiable — CSV schema, duplicate detection logic, and error report format are negotiable
- [x] Valuable — significant time saving for high-volume hiring periods
- [x] Estimable — 3 SP; CSV parsing with validation and batch insert
- [x] Small — import only; requisition management UI is separate
- [x] Testable — import count, error report, column validation, and idempotency are verifiable
