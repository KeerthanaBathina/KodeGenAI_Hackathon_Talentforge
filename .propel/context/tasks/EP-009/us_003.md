# US-003 — Platform Health Dashboard — BullMQ Queue Depth, AI Worker Status, and Email Rate

## User Story

**As a** platform administrator,  
**I want** a dashboard showing BullMQ queue depth, AI worker health, email delivery success rate, and API error rates, refreshed every 60 seconds,  
**So that** operational issues are detected before they impact users.

## Business Value

Proactive operational visibility reduces mean time to detect (MTTD) incidents. Auto-refreshing metrics eliminate the need for engineers to query Grafana for routine health checks.

## Acceptance Criteria

### Scenario 1: Dashboard shows current queue depths
**Given** the admin opens the health dashboard  
**When** the page loads  
**Then** BullMQ queues are listed (resume-parse, email-delivery, screening) with current active, waiting, and failed job counts.

### Scenario 2: AI worker offline shown in amber
**Given** the AI worker has not reported a heartbeat in the last 2 minutes  
**When** the dashboard renders  
**Then** the AI Worker status indicator is amber with text "Last heartbeat: X min ago."

### Scenario 3: Email delivery rate displayed
**Given** 100 emails were attempted in the last hour and 3 failed  
**When** the dashboard refreshes  
**Then** the email delivery success rate shows "97% (3 failed in last 60 min)" and a "View Failed" link opens the dead-letter queue.

### Scenario 4: Dashboard auto-refreshes without full page reload
**Given** the admin is viewing the health dashboard  
**When** 60 seconds pass  
**Then** all metric values update in place (no full page reload); the "Last updated" timestamp reflects the refresh time.

## INVEST Checklist

- [x] Independent — read-only dashboard; does not affect any service
- [x] Negotiable — metric set, refresh interval, and alert thresholds are negotiable
- [x] Valuable — operational awareness reduces incident response time
- [x] Estimable — 5 SP; API polling + metrics display
- [x] Small — observability only; issue remediation is out of scope
- [x] Testable — queue depth accuracy, amber state, email rate, and refresh are verifiable
