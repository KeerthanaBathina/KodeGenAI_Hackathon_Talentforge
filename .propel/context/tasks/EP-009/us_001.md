# US-001 — User Management — Create, Deactivate, and Role Assignment

## User Story

**As a** system administrator,  
**I want** to create new platform users, assign them roles, and deactivate accounts without deleting records,  
**So that** access control is centrally managed and historical activity is preserved when employees leave.

## Business Value

Central user management prevents role creep and ensures only authorised personnel can access sensitive hiring data. Deactivation without deletion preserves audit trail integrity.

## Acceptance Criteria

### Scenario 1: New user created and receives onboarding email
**Given** an admin fills in the create user form with name, email, and role  
**When** they submit  
**Then** a user account is created with `status = "active"`, a temporary password is set, and an onboarding email is sent to the new user within 2 minutes.

### Scenario 2: User deactivated cannot log in
**Given** an admin deactivates a user account  
**When** the deactivated user attempts to log in  
**Then** the login returns "Your account has been deactivated — contact your administrator" and no session is created.

### Scenario 3: Role assignment takes effect on next login
**Given** a user with role `recruiter` is changed to `hr`  
**When** they log out and log in again  
**Then** their JWT contains `role = "hr"` and they are redirected to the HR dashboard.

### Scenario 4: Admin cannot deactivate their own account
**Given** an admin is logged in  
**When** they attempt to deactivate their own user record  
**Then** the API returns HTTP 403 "Administrators cannot deactivate their own account."
