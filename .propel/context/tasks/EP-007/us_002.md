# US-002 — Final Decision Submission — Offer, Reject, Hold, and Withdraw Outcomes

## User Story

**As a** hiring manager,  
**I want** to submit a final hiring decision (offer, reject, hold, withdraw) with a mandatory reason code and optional narrative, and have a decision summary PDF generated automatically,  
**So that** each outcome is documented, communicated, and stored for compliance purposes.

## Business Value

A structured decision record with PDF summary creates a legally defensible, retrievable hiring rationale. The four outcome types cover all real-world scenarios, preventing workaround use of incorrect status values.

## Acceptance Criteria

### Scenario 1: Offer decision triggers approval workflow
**Given** a hiring manager submits an "Offer" decision  
**When** it is saved  
**Then** the application status changes to `"pending_approval"`, the approval chain workflow is initiated (EP-007 US-003), and the hiring manager sees a "Decision submitted — awaiting approval" confirmation.

### Scenario 2: Reject decision sends rejection notification
**Given** a hiring manager submits a "Reject" decision with reason code and optional narrative  
**When** it is saved  
**Then** the application status is `"rejected"`, a rejection email is sent to the candidate within 60 seconds, and a decision summary PDF is generated and stored.

### Scenario 3: Hold decision freezes application without notification
**Given** a hiring manager submits a "Hold" decision  
**When** it is saved  
**Then** the application status changes to `"on_hold"`, no candidate notification is sent, and a reminder task is created for the recruiter in 14 days.

### Scenario 4: Reason code is mandatory before submission
**Given** a hiring manager selects "Reject" without choosing a reason code  
**When** they attempt to submit  
**Then** the submit button remains disabled and inline validation reads "A reason code is required."
