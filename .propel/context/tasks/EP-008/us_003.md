# US-003 — In-App WebSocket Notifications with Badge Count and Toasts

## User Story

**As a** platform user,  
**I want** to receive real-time in-app notifications via WebSocket with a bell-icon badge count and toast pop-ups for important events,  
**So that** I am immediately aware of status changes that require my attention without needing to refresh the page.

## Business Value

Real-time notifications eliminate the need for polling, reduce page-load frequency, and keep all user types responsive to time-sensitive events (SLA alerts, scorecard submissions, offer responses). This directly reduces average response time per event.

## Acceptance Criteria

### Scenario 1: Badge count increments on new notification
**Given** an HR reviewer is logged in and an application enters their review queue  
**When** the WebSocket event fires  
**Then** the notification bell badge increments by 1 within 1 second without a page reload.

### Scenario 2: Toast appears for important events
**Given** a hiring manager's offer approval request has been approved  
**When** the approval event fires  
**Then** a toast notification appears in the top-right corner reading "Offer for [Candidate Name] approved by [Approver Name]" and auto-dismisses after 5 seconds.

### Scenario 3: Notification panel shows history
**Given** a user clicks the notification bell  
**When** the panel opens  
**Then** the last 50 unread and read notifications are listed, grouped by date, with relative timestamps (e.g., "2 min ago").

### Scenario 4: Marking all as read clears badge
**Given** the user has 8 unread notifications  
**When** they click "Mark all as read"  
**Then** all 8 notifications transition to `"read"` state, the badge count drops to 0, and the change persists on page reload.
