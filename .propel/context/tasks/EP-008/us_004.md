# US-004 — Notification Preference Centre with Per-Channel Opt-In/Out

## User Story

**As a** platform user,  
**I want** to manage my notification preferences per channel (email and in-app) for each notification type,  
**So that** I receive only the notifications relevant to my role and do not suffer alert fatigue from excessive communications.

## Business Value

User-controlled notification preferences reduce unsubscribe rates, improve email deliverability metrics, and demonstrate respect for user autonomy — a key GDPR-aligned design principle.

## Acceptance Criteria

### Scenario 1: Preference page lists all notification types per channel
**Given** a user opens Notification Preferences  
**When** the page loads  
**Then** a grid shows each notification type (e.g., "Application Status Change", "Interview Reminder", "Offer Approved") with toggle switches for Email and In-App channels.

### Scenario 2: Opt-out takes effect within 1 send cycle
**Given** a user disables "Interview Reminder" email notifications  
**When** an interview reminder is triggered  
**Then** no reminder email is sent to that user; in-app notification is still sent if that channel remains enabled.

### Scenario 3: System-critical notifications cannot be disabled
**Given** "Account security alerts" is a system-critical notification type  
**When** the user views the preferences grid  
**Then** the toggle for "Account security alerts" is locked (greyed out) with a tooltip "This notification cannot be disabled."

### Scenario 4: Preference changes persist and apply immediately
**Given** a user toggles off an email preference  
**When** they save and log out then log back in  
**Then** the preference is still in the off state and the setting takes effect on the next triggered notification.

## INVEST Checklist

- [x] Independent — preference centre additive layer; does not change delivery logic
- [x] Negotiable — notification type list and locked categories are negotiable

- [x] Small — preference management only; delivery channel wiring is US-002/003
- [x] Testable — preference persistence, opt-out enforcement, and locked fields are verifiable
