# US-001 — Tokenised Email Template Management with Preview and Versioning

## User Story

**As a** platform administrator,  
**I want** to manage all 11 platform email templates with a live preview, version history, and locale fallback,  
**So that** email communications can be updated by the operations team without code changes and previous versions are retrievable for audit.

## Business Value

Editable templates empower the operations team to maintain brand and messaging standards independently. Version history enables rollback when a template update causes issues and satisfies audit requirements for all regulated communications.

## Acceptance Criteria

### Scenario 1: Template editor renders live preview with sample tokens
**Given** an admin opens the `offer_extended` template editor  
**When** they modify the body text  
**Then** the preview pane updates in real time, replacing all `{{token}}` values with sample data (e.g., `{{candidate_name}}` → "Alex Johnson").

### Scenario 2: Saving template creates a new version
**Given** an admin saves a template  
**When** the save completes  
**Then** a new version entry is created in `email_template_versions` with the saved content, `created_by`, and `created_at`; the previous version is retained and retrievable.

### Scenario 3: Version rollback restores previous content
**Given** version 3 of a template introduced an error  
**When** the admin selects version 2 from the version history and clicks "Restore"  
**Then** the active template content reverts to version 2 and a new version 4 record is created capturing the rollback.

### Scenario 4: Missing locale falls back to English
**Given** a French locale template has not been configured  
**When** an email is sent to a French-locale user  
**Then** the English template is used without error and the fallback is logged.
