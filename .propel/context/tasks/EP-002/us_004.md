# US-004 — Application Submission Confirmation and Pre-Review Withdrawal

## User Story

**As a** candidate,  
**I want** to receive an immediate email confirmation when I submit my application and have the option to withdraw it before an HR review begins,  
**So that** I know my application was received and I can correct a mistaken submission without needing to contact HR.

## Business Value

Confirmation emails reduce support requests about "did my application go through." Self-service withdrawal before HR review reduces recruiter workload from cancellation requests and keeps pipeline data clean.

## Acceptance Criteria

### Scenario 1: Confirmation email delivered within 60 s
**Given** a candidate submits their application  
**When** the submission succeeds  
**Then** a confirmation email is sent within 60 seconds containing the role title, company name, application reference ID, and expected next-step timeline.

### Scenario 2: Submission confirmation screen shown immediately
**Given** the candidate clicks "Submit Application"  
**When** the API returns success  
**Then** a confirmation screen is displayed with the application reference ID and a "Track My Application" link.

### Scenario 3: Withdrawal allowed before HR review
**Given** a candidate's application has status `"submitted"` (not yet in HR review)  
**When** they click "Withdraw Application"  
**Then** a confirmation dialog appears; upon confirmation the status changes to `"withdrawn"` and a withdrawal acknowledgement email is sent.

### Scenario 4: Withdrawal blocked once HR review begins
**Given** a candidate's application has status `"under_review"` or later  
**When** they view the application detail  
**Then** the "Withdraw" button is absent; the page shows "Application is under review — contact HR to withdraw."

## INVEST Checklist

- [x] Independent — confirmation/withdrawal does not affect AI screening
- [x] Negotiable — withdrawal window and email content are negotiable
- [x] Valuable — reduces candidate anxiety and HR workload
- [x] Estimable — 5 SP; status machine transition + email trigger
- [x] Small — post-submission state only; form logic is US-002
- [x] Testable — email delivery, status transition, and withdrawal blocking are verifiable
