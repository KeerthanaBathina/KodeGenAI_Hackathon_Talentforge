# US-002 — Shortlist and Reject Decisions with Mandatory Reason Codes

## User Story

**As an** HR reviewer,  
**I want** to shortlist or reject an application by selecting a mandatory reason code and optionally adding a comment,  
**So that** every decision is documented with a business justification that satisfies audit and discrimination-prevention requirements.

## Business Value

Mandatory reason codes create an auditable paper trail that protects the organisation legally and enables data-driven analysis of rejection patterns. They also prevent impulsive decisions without documented rationale.

## Acceptance Criteria

### Scenario 1: Shortlist decision routes application to interview scheduling
**Given** an HR reviewer selects "Shortlist" and chooses reason code "Strong skills match"  
**When** they confirm  
**Then** the application status changes to `"shortlisted"`, the candidate receives a shortlist notification email, and the application appears in the interview scheduling queue.

### Scenario 2: Reject requires reason code selection
**Given** an HR reviewer clicks "Reject" without selecting a reason code  
**When** they attempt to confirm  
**Then** the confirm button remains disabled and an inline message reads "A reason code is required before rejecting."

### Scenario 3: Auto-rejection email sent within 60 s
**Given** a reviewer rejects an application with reason code "Insufficient experience"  
**When** the decision is saved  
**Then** a personalised rejection email using the `application_rejected` template is sent to the candidate within 60 seconds.

### Scenario 4: Decision written to audit trail
**Given** any shortlist or reject decision is submitted  
**When** it is saved  
**Then** an `audit_events` record is created with `actor_id`, `event_type = "application_decision"`, `entity_id`, `reason_code`, and timestamp.
