# US-003 — Automatic Path Classification (Fresher vs. Experienced) with Recruiter Override

## User Story

**As a** recruiter,  
**I want** the system to automatically classify shortlisted candidates into "fresher" or "experienced" interview paths based on years of experience and allow me to override that classification with a justification,  
**So that** each candidate follows the interview structure appropriate for their career stage.

## Business Value

Automatic classification eliminates manual routing decisions for the majority of candidates. The override capability handles edge cases while maintaining an auditable record of every classification change.

## Acceptance Criteria

### Scenario 1: Candidate with < 2 years classified as fresher
**Given** `screenings.parsed_data.experience_years = 1`  
**When** the application is shortlisted  
**Then** `applications.path = "fresher"` is automatically set and the fresher interview sequence is assigned.

### Scenario 2: Candidate with ≥ 2 years classified as experienced
**Given** `screenings.parsed_data.experience_years = 3`  
**When** the application is shortlisted  
**Then** `applications.path = "experienced"` is automatically set and the experienced interview sequence is assigned.

### Scenario 3: Override requires written justification
**Given** a recruiter wants to change a classification from "experienced" to "fresher"  
**When** they click the override button  
**Then** a modal requires a justification text (minimum 20 characters) before confirming; the button is disabled without it.

### Scenario 4: Override written to audit trail
**Given** a recruiter overrides the path classification  
**When** they confirm  
**Then** an `audit_events` record captures `original_path`, `new_path`, `justification`, `actor_id`, and timestamp.

## INVEST Checklist

- [x] Independent — classification runs after shortlist; does not affect screening
- [x] Negotiable — experience threshold (2 years) is a configurable business rule
- [x] Valuable — ensures correct interview structure for each candidate cohort
- [x] Estimable — 5 SP; conditional logic + override modal
- [x] Small — routing classification only; interview stages are EP-005
- [x] Testable — path value, threshold behaviour, and justification requirement are verifiable
