# US-004 — Bulk Reject Action and Real-Time Queue Notifications via WebSocket

## User Story

**As an** HR manager,  
**I want** to select multiple applications and bulk-reject them with a shared reason code and receive real-time WebSocket notifications when new applications enter the review queue,  
**So that** I can process large batches efficiently and respond immediately to new arrivals without manual page refreshes.

## Business Value

Bulk actions reduce repetitive individual decision clicks when clearing a backlog of clearly-below-threshold applications. Real-time notifications keep queue counts accurate without polling overhead.

## Acceptance Criteria

### Scenario 1: Bulk select and reject submits all in one action
**Given** an HR reviewer selects 5 applications using the row checkboxes  
**When** they click "Bulk Reject" and select a shared reason code  
**Then** all 5 applications transition to `"rejected"` and 5 rejection emails are queued in a single API call.

### Scenario 2: Bulk action requires at least 2 applications selected
**Given** only 1 application is selected  
**When** the HR reviewer clicks "Bulk Reject"  
**Then** the button is disabled with a tooltip "Select at least 2 applications for bulk action."

### Scenario 3: Real-time badge updates on new queue entry
**Given** an HR reviewer is on the review queue page  
**When** a new application is screened and enters the queue  
**Then** the queue navigation badge increments within 1 second (WebSocket push); a toast notification appears reading "New application: [Candidate Name] for [Role]."

### Scenario 4: Queue count is accurate after bulk rejection
**Given** the reviewer performs a bulk reject of 5 applications  
**When** the action completes  
**Then** the queue badge count decreases by 5 and the rejected rows are removed from the view without a full page reload.
