# US-005 — Interview Path Enforcement — Fresher Stage Prerequisites and Experienced Path Gating

## User Story

**As a** recruiter,  
**I want** the system to enforce that candidates complete each interview stage in the correct order for their path (fresher: aptitude → technical → cultural; experienced: technical → system-design → cultural) before the next stage can be scheduled,  
**So that** candidates cannot skip stages and the interview data is always complete and sequentially valid.

## Business Value

Enforced stage sequencing ensures every hiring decision is based on a complete set of structured assessments. Skipped stages create gaps in the data that undermine the fairness and defensibility of hiring outcomes.

## Acceptance Criteria

### Scenario 1: Fresher candidate cannot schedule technical before aptitude
**Given** a fresher candidate has not completed the aptitude stage  
**When** the recruiter attempts to schedule the technical interview  
**Then** the scheduling button is disabled and a tooltip reads "Aptitude stage must be completed before scheduling Technical."

### Scenario 2: Fresher path stages complete in order
**Given** a fresher candidate has completed aptitude and technical stages  
**When** the recruiter opens the scheduling UI  
**Then** only the cultural fit stage is available to schedule; aptitude and technical are shown as completed.

### Scenario 3: Experienced path skips aptitude stage
**Given** a candidate is classified as "experienced"  
**When** they are shortlisted  
**Then** the aptitude stage is absent from their interview sequence; only technical, system design, and cultural fit stages are present.

### Scenario 4: API enforces stage prerequisites server-side
**Given** the frontend stage gate is bypassed (e.g., direct API call)  
**When** a POST to `/interviews` is made for a stage whose prerequisite is incomplete  
**Then** the API returns HTTP 422 "Prerequisite stage [name] is not complete" and no interview record is created.

## INVEST Checklist
