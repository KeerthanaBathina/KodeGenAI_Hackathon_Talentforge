# US-003 — Scorecard Capture with Mandatory Rubric Dimensions and Recommendation Submission

## User Story

**As a** panel interviewer,  
**I want** to complete a structured scorecard with pre-defined rubric dimensions for each interview stage and submit my recommendation (advance, hold, reject),  
**So that** interview feedback is standardised, comparable across candidates, and captured before I leave the interview session.

## Business Value

Standardised rubric scoring eliminates interviewer-to-interviewer inconsistency and enables statistical analysis of hiring panel decisions. Mandatory capture before leaving the session prevents feedback decay.

## Acceptance Criteria

### Scenario 1: Scorecard loads with stage-appropriate rubric
**Given** an interviewer opens a technical interview scorecard  
**When** the form loads  
**Then** rubric dimensions specific to "Technical" stage (Problem Solving, Code Quality, System Design, Communication) are displayed; a cultural-fit interview shows its own distinct dimensions.

### Scenario 2: All rubric dimensions must be scored before submitting
**Given** an interviewer has scored 3 of 4 rubric dimensions  
**When** they click "Submit Scorecard"  
**Then** the submit button is disabled and an inline message highlights the unscored dimension.

### Scenario 3: Scorecard submitted and locked
**Given** all rubric dimensions are scored and a recommendation is selected  
**When** the interviewer clicks "Submit"  
**Then** the scorecard status changes to `"submitted"`, the form becomes read-only, and the aggregate panel score is recalculated.

### Scenario 4: Partial save allows return before submission
**Given** an interviewer has scored 2 of 4 dimensions  
**When** they close the tab and return later  
**Then** their 2 scored dimensions are pre-filled and they can continue from where they left off.

## INVEST Checklist

- [x] Independent — scorecard independent of decision workflow
- [x] Negotiable — rubric dimensions and weight per dimension are negotiable
- [x] Valuable — ensures structured, auditable interview feedback
- [x] Estimable — 8 SP; multi-dimension form with partial save and lock
- [x] Small — scorecard capture only; decision workflow is EP-007
- [x] Testable — mandatory fields, partial save, and lock behaviour are verifiable
