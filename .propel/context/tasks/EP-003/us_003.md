# US-003 — AI Screening Score Computation with Configurable Thresholds

## User Story

**As an** HR manager,  
**I want** the AI to compute a match score (0–100) for each application against the requisition requirements, identifying positive factors and skill gaps, using configurable shortlist and borderline thresholds,  
**So that** the screening step is objective, consistent, and adjustable without code changes.

## Business Value

AI-based screening reduces unconscious bias and ensures every application is evaluated against the same criteria. Configurable thresholds allow HR to tune precision vs. recall based on hiring goals without engineering involvement.

## Acceptance Criteria

### Scenario 1: Score computed and written within 60 s of parsing
**Given** resume parsing completes  
**When** the screening job is enqueued  
**Then** the AI screening score is written to `screenings.score` within 60 seconds.

### Scenario 2: Score above threshold creates shortlist recommendation
**Given** the shortlist threshold is set to 75  
**When** an application scores 80  
**Then** `screenings.recommendation = "shortlist"` is stored and the application advances to the HR review queue automatically.

### Scenario 3: Score below reject threshold creates reject recommendation
**Given** the reject threshold is set to 40  
**When** an application scores 35  
**Then** `screenings.recommendation = "reject"` is stored and an auto-rejection email is queued.

### Scenario 4: Score in borderline range flags for manual review
**Given** the borderline range is 40–75  
**When** an application scores 55  
**Then** `screenings.recommendation = "manual_review"` is stored and the application appears in the HR manual review queue.

### Scenario 5: Threshold change applies only to new screenings
**Given** the shortlist threshold is changed from 75 to 80  
**When** applications submitted before the change are already screened  
**Then** their scores and recommendations are unchanged; new applications use the updated threshold.
