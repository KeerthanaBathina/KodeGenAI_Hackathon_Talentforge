# US-005 — Low-Confidence Escalation to Manual Review and AI Fallback Mode

## User Story

**As an** HR manager,  
**I want** applications with low AI confidence automatically escalated to a manual review queue and the platform to continue accepting applications with a human-review path when the AI worker is unavailable,  
**So that** no application is lost or delayed due to AI uncertainty or infrastructure issues.

## Business Value

Automated escalation ensures borderline candidates receive human consideration. A fallback mode guarantees platform availability and candidate SLA compliance even during AI service outages.

## Acceptance Criteria

### Scenario 1: Low-confidence application escalated to manual queue
**Given** a screening result has `confidence < 0.5` (regardless of score)  
**When** the screening job completes  
**Then** the application is added to the HR manual review queue with a "Low AI Confidence" badge and no automated recommendation is applied.

### Scenario 2: Manual review queue is reachable without AI screening
**Given** the AI worker queue depth exceeds 100 or the worker has been offline for > 5 minutes  
**When** a candidate submits an application  
**Then** the system enters fallback mode: applications are placed directly in the manual review queue with status `"pending_manual_review"` and the AI screening step is bypassed.

### Scenario 3: Fallback mode banner shown to HR users
**Given** fallback mode is active  
**When** an HR user opens the review dashboard  
**Then** an amber banner reads "AI screening temporarily offline — applications are routed to manual review."

### Scenario 4: AI worker recovery clears fallback mode automatically
**Given** fallback mode is active  
**When** the AI worker is responsive and queue depth drops below 20  
**Then** fallback mode is deactivated, the banner disappears, and new applications are screened by AI again.

## INVEST Checklist

- [x] Independent — fallback mode independent of screening algorithm
- [x] Negotiable — confidence threshold, queue-depth trigger, and banner text are negotiable
- [x] Valuable — platform resilience; protects candidate SLA
