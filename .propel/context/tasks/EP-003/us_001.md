# US-001 — Secure Resume Upload via Presigned URL with Malware Scanning

## User Story

**As a** candidate,  
**I want** to upload my resume (PDF or DOCX, up to 10 MB) directly to secure storage using a presigned URL,  
**So that** my file is stored safely without the API server having access to the file contents, and it is scanned for malware before being processed.

## Business Value

Direct-to-storage upload via presigned URLs eliminates API server exposure to raw file contents, reducing attack surface. Malware scanning protects the AI parsing worker and HR users from malicious files.

## Acceptance Criteria

### Scenario 1: Presigned URL issued and upload completes
**Given** a candidate is on the resume upload step  
**When** they select a valid PDF file ≤ 10 MB  
**Then** the API issues a presigned URL, the browser uploads directly to Supabase Storage, and the UI shows an upload progress bar.

### Scenario 2: File type and size validated before upload
**Given** a candidate selects a `.exe` file or a PDF over 10 MB  
**When** client-side validation runs  
**Then** an error message is shown ("Only PDF and DOCX files up to 10 MB are accepted") before any upload attempt occurs.

### Scenario 3: Malware scan quarantines infected file
**Given** a file fails the ClamAV/anti-malware scan  
**When** the scan result arrives via webhook  
**Then** the file is moved to a quarantine bucket, the resume status is set to `"quarantined"`, and the candidate is notified with "Your file could not be accepted — please upload a different file."

### Scenario 4: Clean file triggers AI parsing queue
**Given** a file passes the malware scan  
**When** scan completes  
**Then** the file status changes to `"queued_for_parsing"` and a job is enqueued in BullMQ for the AI parser.
