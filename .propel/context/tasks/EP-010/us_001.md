# US-001 — Pipeline Dashboard with KPI Metrics and Per-Requisition Filter

## User Story

**As a** recruiting manager,  
**I want** a pipeline dashboard showing total applications, shortlist rate, time-to-hire, and offer acceptance rate — filterable by requisition — updated with no more than 5 minutes of lag,  
**So that** I can monitor hiring health at a glance and identify bottlenecks before they affect delivery targets.

## Business Value

Real-time KPI visibility replaces weekly manual spreadsheet reporting. Identifying shortlist rate drops or time-to-hire increases within 5 minutes of occurrence enables immediate corrective action.

## Acceptance Criteria

### Scenario 1: Dashboard KPIs load within 2 s
**Given** the recruiting manager navigates to the pipeline dashboard  
**When** the page loads  
**Then** total applications, shortlist rate, average time-to-hire, and offer acceptance rate are all displayed within 2 seconds.

### Scenario 2: Per-requisition filter updates all KPIs
**Given** the manager selects "Senior Backend Engineer – London"  
**When** the filter is applied  
**Then** all four KPIs recalculate to show values only for that requisition's pipeline.

### Scenario 3: Data lag does not exceed 5 minutes
**Given** an application is submitted at 10:00 AM  
**When** the manager views the dashboard at 10:05 AM  
**Then** the application is reflected in the total applications count.

### Scenario 4: Time-to-hire excludes withdrawn and draft applications
**Given** some applications have status `"withdrawn"` or `"draft"`  
**When** average time-to-hire is computed  
**Then** only applications that reached `"offer_accepted"` are included in the calculation.
