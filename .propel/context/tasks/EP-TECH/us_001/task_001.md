# TASK-001 — Scaffold Next.js 14 App Router Project with TypeScript Strict Mode

## Context

**User Story**: US-001 — Deploy Frontend to Vercel with CDN and Preview Environments  
**Epic**: EP-TECH — Technical Bootstrap  
**Addresses Acceptance Criteria**: Scenario 3 (build failure blocks deployment), Scenario 4 (environment variables injected at build time)

This task establishes the Next.js 14 project skeleton that all downstream frontend feature epics (EP-001 through EP-011) will build upon. The scaffold must compile cleanly and serve as the contractual build artefact that Vercel deploys.

---

## Objective

Initialise a Next.js 14 (App Router) project with TypeScript strict mode, configure security-relevant headers, and define the `.env.example` contract so that environment variable injection is explicit and auditable.

## Implementation Steps

### Step 1 — Bootstrap the project

```bash
npx create-next-app@14 frontend \
  --typescript \
  --tailwind \
  --eslint \
  --app \
  --src-dir \
  --import-alias "@/*" \
  --use-npm
cd frontend
```

> **Note**: Use `--app` flag to opt into the App Router. Do not use Pages Router.

### Step 2 — Enforce TypeScript strict mode

Verify `tsconfig.json` contains the following (generated by default, but confirm):

```json
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "forceConsistentCasingInFileNames": true,
    "noEmit": true,
    "target": "ES2022",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": false,
    "skipLibCheck": true,
    "esModuleInterop": false,
    "allowSyntheticDefaultImports": true,
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "incremental": true,
    "plugins": [{ "name": "next" }],
    "paths": { "@/*": ["./src/*"] }
  }
}
```

### Step 3 — Configure `next.config.js` with security headers

Create `frontend/next.config.js`:

```js
/** @type {import('next').NextConfig} */
const securityHeaders = [
  { key: 'X-DNS-Prefetch-Control', value: 'on' },
  { key: 'Strict-Transport-Security', value: 'max-age=63072000; includeSubDomains; preload' },
  { key: 'X-Frame-Options', value: 'SAMEORIGIN' },
  { key: 'X-Content-Type-Options', value: 'nosniff' },
  { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
  { key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=()' },
  {
    key: 'Content-Security-Policy',
    value: [
      "default-src 'self'",
      "script-src 'self' 'unsafe-eval' 'unsafe-inline'",
      "style-src 'self' 'unsafe-inline'",
      "img-src 'self' data: blob:",
      "font-src 'self'",
      "connect-src 'self' https://*.supabase.co https://*.railway.app",
      "frame-ancestors 'none'",
    ].join('; '),
  },
];

const nextConfig = {
  async headers() {
    return [{ source: '/(.*)', headers: securityHeaders }];
  },
  poweredByHeader: false,
};

module.exports = nextConfig;
```

### Step 4 — Define environment variable contract

Create `frontend/.env.example`:

```env
# Backend API base URL — required at build time for Next.js public env vars
# Set this value in Vercel project settings; do NOT commit the real value
NEXT_PUBLIC_API_URL=https://your-api.railway.app

# Supabase public keys — safe to expose in browser
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

Create `frontend/.env.local` (do not commit — add to `.gitignore`):

```env
NEXT_PUBLIC_API_URL=http://localhost:3001
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

Update `frontend/.gitignore` to include:

```
.env.local
.env.*.local
```

### Step 5 — Verify clean build locally

```bash
cd frontend
npm install
npm run type-check    # tsc --noEmit
npm run build         # next build
```

Expected: zero TypeScript errors, zero ESLint errors, successful `.next` output.

### Step 6 — Add `type-check` script to `package.json`

```json
{
  "scripts": {

    "lint": "next lint",

  }
}
```

## Security Constraints

- **OWASP A05 (Security Misconfiguration)**: Security headers configured in `next.config.js` (CSP, HSTS, X-Frame-Options, X-Content-Type-Options).
- **OWASP A02 (Cryptographic Failures)**: `NEXT_PUBLIC_*` variables are safe to expose in browser; no server secrets prefixed with `NEXT_PUBLIC_`.
- `poweredByHeader: false` suppresses the `X-Powered-By: Next.js` fingerprint.
