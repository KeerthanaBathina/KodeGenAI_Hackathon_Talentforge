# US-001 — Deploy Frontend to Vercel with CDN and Preview Environments

## User Story

**As a** developer,  
**I want** the React/Next.js frontend deployed to Vercel with automatic preview URLs on every pull request,  
**So that** stakeholders can review UI changes in a live environment before merge and production traffic is served from a global CDN.

## Business Value

Zero-configuration CDN hosting eliminates infrastructure management and ensures sub-100ms global response times. Preview environments reduce review cycle time from hours to minutes.

## Acceptance Criteria

### Scenario 1: Production deployment on main merge
**Given** a commit is merged to `main`  
**When** the GitHub Actions CD workflow triggers  
**Then** the build completes and the Vercel production URL returns HTTP 200 within 3 minutes.

### Scenario 2: Preview URL generated for every PR
**Given** a pull request is opened  
**When** CI runs  
**Then** a unique preview URL is posted as a PR comment within 5 minutes.

### Scenario 3: Build failure blocks deployment
**Given** a TypeScript compile error exists in the PR branch  
**When** CI runs  
**Then** deployment is blocked and the PR merge button is disabled.

### Scenario 4: Environment variables injected at build time
**Given** `NEXT_PUBLIC_API_URL` is stored in Vercel project settings  
**When** the build runs  
**Then** the value is available at runtime and not exposed in the build artifact source map.
