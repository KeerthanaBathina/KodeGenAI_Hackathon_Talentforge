# US-004 — Forgot Password with Time-Limited Reset Link

## User Story

**As a** platform user,  
**I want** to receive a one-time password-reset link that expires in 60 minutes,  
**So that** I can regain account access securely without revealing whether my email is registered.

## Business Value

A secure, time-limited reset flow reduces the support burden of locked accounts while preventing user enumeration. Short token expiry limits the window of exposure if an email is intercepted.

## Acceptance Criteria

### Scenario 1: Reset link delivered within 60 s
**Given** a registered user enters their email on the forgot-password page  
**When** they submit  
**Then** a password-reset email containing a unique signed link is delivered within 60 seconds.

### Scenario 2: Non-enumeration response for unknown email
**Given** an email address that does not exist in the system  
**When** the reset form is submitted  
**Then** the UI shows the same success message as a real email ("If this email is registered, you will receive a reset link") and no 404 or user-not-found indication is returned.

### Scenario 3: Reset link expires after 60 minutes
**Given** a user clicks a reset link 61 minutes after it was issued  
**When** the link is opened  
**Then** the page shows "This link has expired — please request a new one" and the token cannot be used to change the password.

### Scenario 4: Token is single-use
**Given** a user successfully resets their password  
**When** they attempt to use the same reset link a second time  
**Then** the link is rejected with "This link has already been used" and the token is invalidated in the database.

## INVEST Checklist

- [x] Independent — password reset independent of SSO and application features
- [x] Negotiable — token expiry window is negotiable (30–120 min)
- [x] Valuable — critical for user recovery without support intervention
- [x] Estimable — 3 SP; standard Supabase Auth reset pattern
- [x] Small — single flow; no profile or session logic
- [x] Testable — expiry, single-use, and non-enumeration are verifiable
